# Спека — слой «продукт / маэстро» + `ccmusician`

> Дизайн от 2026-06-30. Поднимаем харнесс с «один музыкант на репо» до «маэстро водит
> продукт из нескольких репо». Решения F1–F4 зафиксированы; стройки ещё нет.

## Зачем

Сейчас музыкант доводит один кусок работы на одном репо до конца. Нужен слой выше: **маэстро
водит продукт** — набор репозиториев (фронтенд / бэкенд / …), заводя по музыканту на репо. Музыкант
остаётся как есть. Добавляются: CLI на одного музыканта (`ccmusician`), мозг-маэстро (оркестратор
продукта), модель продукта + продуктовый роадмап. Сверху — hermes (человек-надзиратель).

## Слои

- **funnel** — как строится один кусок работы (есть)
- **музыкант** — один кусок на одном репо до конца (есть)
- **nonstop** — гонит музыканта по роадмапу репо соло, перезапуская после каждого закрытия (есть)
- **маэстро (новое)** — водит продукт = набор репо: какие активны, по музыканту на репо, порядок
  связанных меж-репо фич
- **hermes** — человек-надзиратель над маэстро

Параллельность = **один музыкант на репо** (репозитории разные → нет драки за общее дерево и за
вливание в `main`). «Много музыкантов в одном репо» — осознанно вне схемы.

## Зафиксированные решения

- **F1 — `ccmusician` отдельно в cc-agent.** Standalone, ноль зависимостей; одно-репному человеку
  хватает только cc-agent (без cc-maestro). Унификация с `ccmaestro` — общая **грамматика команд по
  договорённости**, не общий код; дублирование принято осознанно. `ccmusician` = выросшая bash-утилита
  `musicians`.
- **F2 — продуктовый роадмап федеративный.** Репо-роадмапы остаются как есть; продуктовый слой
  добавляет только реестр репо + рёбра меж-репо порядка. Не мастер-список с декомпозицией по репо.
- **F3 — мозг-маэстро гоняется через `/loop`** (проход раз в N часов), а не через свой Stop-хук.
- **F4 — bash `musicians` поглощается `ccmusician`** (переименование/расширение); отдельного fallback
  нет.

## Унифицированная грамматика CLI (контракт, который соблюдают оба)

Общие подкоманды — у `ccmusician` scope = текущий репо, у `ccmaestro` = машина / продукт:

- `ls [--json]`
- `info <id>` (для музыканта — детальная карточка)
- `logs <id> [--tail N]`
- `watch <id>`
- `start "<task>" [--repo P]`
- `stop <id>` (музыкант → мягкий cancel)
- `nonstop <id> on|off`

Только `ccmaestro`: `product`, `check [--notify]`, `pause` / `resume`, `steer`, вердикты вотчдога.

## Работа по порядку зависимостей

**W0. Контракт грамматики.** Зафиксировать общий набор подкоманд (выше) как то, что оба CLI обязаны
соблюдать. Лёгкий шаг — документ, без общего кода (F1).

**W1. `ccmusician` (cc-agent, bash, ноль зависимостей).** Вырастить `musicians` в полноценный CLI на
одного музыканта: `ls / info / watch / logs / stop / nonstop` + новый `start` (поднять музыканта на
текущем репо через `claude -p "/musician …"`). Scope = текущий репо. Даёт «hermes рулит музыкантом
напрямую на одном репо» сразу, ещё до мозга-маэстро. `musicians` оставить как алиас.

**W2. Продукт + продуктовый роадмап.** Продукт = реестр репо + тонкий продуктовый North Star + рёбра
порядка («фича X в репо A раньше фичи Y в репо B»). Живёт вне отдельного репо, чтобы маэстро его видел
(кандидат — файл рядом с конфигом ccmaestro, `~/.ccmaestro/products/<name>`). Команда `ccmaestro
product` читает/правит.

**W3. Мозг-маэстро (cc-maestro, скилл под `/loop`).** Один проход: прочитать продуктовый роадмап →
`ccmaestro ls --json` → на каждом репо с незакрытой фичей и без живого музыканта завести музыканта /
включить nonstop → выдержать меж-репо порядок (придержать репо B, пока в A не сел нужный коммит) →
отчитаться. Опирается на nonstop для соло-хождения по каждому репо; уникальное — выбор активных репо,
порядок связанных фич, здоровье.

**W4. Унификация + обвязка.** Подровнять грамматику `ccmaestro` под контракт W0, тесты, доки, версии
(cc-agent — за `ccmusician`; cc-maestro — за продукт + мозг).

## Что переиспользуется (уже готово)

- `ccmaestro start --repo <путь>` — завести агента в конкретном репо
- `ccmaestro ls --json` / `ccmaestro musician --json` — наблюдение
- `ccmaestro stop` — мягкий cancel музыканта
- слой **nonstop** — соло-хождение по роадмапу репо
- bash `musicians` — основа под `ccmusician`

## Самый твёрдый кусок

W3, **меж-репо порядок связанных фич** (контракт API: бэкенд раньше фронта) — единственное место, где
маэстро реально «думает», а не просто заводит музыкантов. Всё остальное — переиспользование готового +
унификация.
